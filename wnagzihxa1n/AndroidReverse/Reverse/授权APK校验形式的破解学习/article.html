跟入checkappKey()函数

再跟入getAppKey()函数

这里就有个比较有意思的知识点了，通常也不是经常碰到，这里碰到了就仔细的讲讲

private String getAppKey(int resId) {
    String result = "";
    try {
        return createPackageContext("com.droider.appkey", 2).getString(resId);
    } catch (Exception e) {
        e.printStackTrace();
        return "";
    }
}

首先我们注意本应用是一个代表功能型的应用，它需要额外安装不同的授权APK，才能使用不同的功能

我们来看它的包名

com.droider.free

也就是说，该应用会在启动的时候，先检测一下是否有安装com.droider.appkey这个用于授权的APK应用，如果存在，则获取传入的那个资源ID2130903041指向的String类型资源，一开始我这里没有太注意，直接去R资源文件搜，难怪搜不到

这里从授权APK里获取了一个String类型的字符串

获取到字符串就返回True，进入中间获取resID的代码段，否则直接赋值R.string.free

if (checkappKey()) {
    resID = decryptAppKey(getAppKey(2130903041));
    if (resID == 0) {
        resID = R.string.free;
    }
} else {
    resID = R.string.free;
}

关键的就是进入decryptAppKey(getAppKey(2130903041))函数

它中间又调用crypt类的decrypt()函数，然后将结果转为int类型返回，返回的结果赋值给上一层函数的redID

跟入crypt.decrypt()

那么它有两个参数，第一个参数已确定，是"droider"，第二个参数由授权APK确定

那么中间又有几个函数，我们分开分析

return new String(decrypt(getRawKey(seed.getBytes()), toByte(encrypted)));

new String()里面是一个整体的函数

decrypt(getRawKey(seed.getBytes()), toByte(encrypted))

decrypt()里是两个参数，第一个通过函数调用获取返回值当参数，第二个看起来像是获取byte[]

先来第一个参数，这是在生成AES加密的KEY

再往回退，可以看到，那个decrypt()的两个参数就确定了，前面是解密的KEY，后面是待解密的数据

那么大概就是AES解密了，解密出来的数据是一个int型的resID

整个加解密代码已经在样本里有了

回到最开始的入口，此时我们已经弄清楚这里是在处理授权，会进行一个AES解密操作，解密出一个int型的resID，在下面会进行对应字符串的获取，然后对比

根据代码的对比，我在对应的资源中提取出了下面这些东西

public static final int free = 2130968577;
public static final int advanced = 2130968578;
public static final int pro = 2130968579;

分别对应

Android安全软件免费版
Android安全软件高级版
Android安全软件专业版

那么分析到这里，整个校验授权的逻辑就已经很清楚了，先获取到授权APK的一个加密字符串，然后使用AES解密，得到一个int型的resID，根据这个resID进行判断，程序会决定显示出多少个按钮可见

if (resID == R.string.advanced) {
    this.btn_advanced.setVisibility(0);
} else if (resID == R.string.pro) {
    this.btn_advanced.setVisibility(0);
    this.btn_pro.setVisibility(0);
}

0x02 多方式破解

2.1 直接爆破

这种方式就很粗暴了，不过我喜欢，首先是第一个校验点checkappKey()函数，它在没有检测到有授权APK存在的时候就会返回false，resID为R.string.free

if (checkappKey()) {
    resID = decryptAppKey(getAppKey(2130903041));
    if (resID == 0) {
        resID = R.string.free;
    }
} else {
    resID = R.string.free;
}

我们在这个地方，可以有三种改法

  • 删除这段代码，直接将resID赋值为R.string.advanced/R.string.pro对应的resID
  • 第一个if判断取反，然后在里面进行resID的赋值
  • 修改else分支的代码，给resID赋值

还是第三种改法好，至于改成高级还是专业，那么谁跟专业版过不去呢？
