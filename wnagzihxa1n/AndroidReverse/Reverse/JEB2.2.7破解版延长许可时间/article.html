那这个就比较尴尬了，现在是2016.10.24，再怎么算也就四个月了哇，在这四个月的时间里出泄露出新版本的JEB目测有点悬，所以还是自己动手把这个版本的许可证延一下期好了

首先一开始我是不知道这个校验在哪个位置的，因为2.x和1.x的文件目录不一样，1.x的jeb.jar直接在bin文件夹下面，而2.x的jeb.jar在bin/plugins下面的com.pnfsoftware.jeb.rcpclient_2.2.7.201608151620.jar文件里面，使用jdgui打开的话可以很明显的看到，至于如何知道校验在这个文件里？首先是这个文件比较大，代码肯定比较多，然后名字里面有jeb，所以猜了一波，运气也是比较好，一下就猜中了，猜不中就慢慢找吧

jdgui打开com.pnfsoftware.jeb.rcpclient_2.2.7.201608151620.jar文件后，使用搜索功能，搜索上面那个Support expires on字符串，可以看到只有一处，那么肯定就是校验的地方了，或者说是调用校验的地方了

双击跟过去

可以看到先是获取一个数值赋值给i，这个i在后面还会用于显示日期，那么下面这个方法至关重要

Licensing.getExpirationTimestamp()

跟过去，实现的代码比较简单

public static final int getExpirationTimestamp()
{
    if (real_license_ts <= 0) {
        return 0;
    }
    return real_license_ts + 31536000;
}

31536000经常搞破解的同学应该比较熟悉，就是365天，一年，那么隐约可以感觉到这个的意思是许可证只有一年的有效期

那么real_license_ts是什么呢？

我们来看开头定义的一些变量

可以看到real_license_ts的值是1456790400，我们转换一下，大约是46.1945年，我们知道我们在进行日期计算或者取值的时候，基本日期是1970年，那么这里就可以猜到这是注册日期或者说是许可证生效的日期，大约是2016年的样子，结合我们原版的到期时间应该是2016年3月1号，然后刚才那个是计算许可证失效日期，大概就是这么个意思

那么现在可以思考一下，如何破解？

一个是修改它返回的日期，另一个就是把检测许可证失效的那个方法去掉，当然这只是猜想，可以试一试

有了大概的思路，我们来实现一下

因为这是一个jar文件，所以直接修改的话不太现实，jar本质上是一个压缩包，里面是各种class文件

这里我们采用javassit来修改，javassit也是一个jar包，在官方下载的文件夹里自带了一部分的sample，可以自己去读一读代码看看各种实现

这里我用修改jeb.jar里Licensing.class文件的getExpirationTimestamp()方法来做一个简单的演示

首先是创建一个Java工程，Eclipse就可以，导入javassit.jar文件

代码如下

import javassist.*;

public class Main {
    public static void main(String[] args) throws Exception {
        ClassPool pool = ClassPool.getDefault();
        pool.insertClassPath("E:\\workplace\\JEBCrack\\jeb.jar");
        CtClass ctClass = pool.get("com.pnfsoftware.jeb.client.Licensing");
        try{
            CtMethod ctMethod = ctClass.getDeclaredMethod("getExpirationTimestamp");
            ctMethod.setBody("return 2000000000;");
            ctClass.writeFile("E:\\");
        }catch(Exception e){
            e.printStackTrace();
        }
    }
}

讲一下代码的意思：首先获取ClassPool，这个是池的意思，如果路径没有这个jeb.jar的话就需要下面进行插入，也就是insertClassPath()方法，把你的jeb.jar完整路径修改一下，接着获取class，这里就是Licensing.class，路径要完整，接着就是获取方法，我们直接将方法的Body设置为返回2000000000，这个数值足够大而不会导致int溢出，最下面一句是修改后的文件保存的路径，所以运行起来后会在E盘下面有一个com文件夹，将里面的Licensing.class文件替换掉最开始那个com.pnfsoftware.jeb.rcpclient_2.2.7.201608151620.jar文件里的Licensing.class文件即可
